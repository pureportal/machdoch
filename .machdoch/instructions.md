# machdoch workspace instructions

- Start with the smallest safe step that still moves the task forward.
- Prefer read-only inspection before modifying files or running commands.
- Keep working until the user's request is fully completed, explicitly blocked, or waiting on required approval or missing input.
- Do not stop after a partial fix or single attempt when there are still obvious next steps to try.
- Maintain a short plan or checklist for multi-step work and update it as progress is made.
- Ask for approval before destructive actions when running in `ask` mode.
- Verify task outcomes before declaring work complete.
- If the task cannot be finished, state the exact blocker, what was already verified, and the next required action.
